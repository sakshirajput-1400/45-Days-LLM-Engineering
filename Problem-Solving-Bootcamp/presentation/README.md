# Speaker Notes — *Why You Freeze at a Blank File* (15 slides, ~30 min)

Open [`index.html`](index.html) in any browser. `→`/`Space` to advance, `F` for fullscreen.

**The one message of this deck:** reading code trains *recognition*, writing code requires
*generation* — they are different muscles, and students have hundreds of reps of the first and
almost none of the second. Everything else is the practice routine that fixes it.

| # | Slide | What to say / do (timing) |
|--:|-------|---------------------------|
| 1 | Cover | Read the title out loud and ask **"who has felt exactly this?"** — every hand goes up. That show of hands is the whole hook: it's not a personal failing, it's universal. (2 min) |
| 2 | The symptom | Let them recognize themselves. Emphasize the reframe: *instant understanding of solutions proves comprehension is fine* — so whatever is broken, it isn't intelligence. (2 min) |
| 3 | Two muscles | The core slide — go slow. Recognition: every line arrives with context, you only verify. Generation: blank file, you must decompose + recall + sequence. Punchline: "100 solutions watched = 100 recognition reps, **0** generation reps." (3 min) |
| 4 | Illusion of competence | The cricket analogy lands hardest here — watching Kohli doesn't train your cover drive. Also the exam parallel: re-reading a chapter vs producing on a blank answer sheet. (2 min) |
| 5 | Tutorial-hell loop | Walk the loop left to right. Key sentence: **"the feelings lie"** — struggle feels like failure, peeking feels like progress, so students optimize for the wrong feeling. Every peek = one skipped rep. (2 min) |
| 6 | The hidden hard part | Show that a solution file is only Job 2. Job 1 (decomposition) was done invisibly by the author — that's *why* solutions read so smoothly. Their gap is Job 1, not Python. (3 min) |
| 7 | The fix | Redefine the rep: minutes stuck/planning/trying count, lines read don't. Mention productive-struggle research: struggling *first* makes the same solution teach more. (2 min) |
| 8 | Rule 1 — plan first | Tie to the Day-5 method if the cohort knows it. Key line: "if you can't write the plan, you found the real gap — and it wasn't syntax." (2 min) |
| 9 | Rule 2 — struggle timer | Walk the minute-by-minute feelings so they know what's normal. Then have the whole room actually say the action line together: **"Stuck is the rep."** Corny, works. (2 min) |
| 10 | Rule 3 — hint ladder | Explain the 3 rungs and the discipline: one rung, then back to trying. Point out every problem today has a `HINTS.md` built exactly like this. (2 min) |
| 11 | Rule 4 — own it | Reading solutions is allowed! The crime is stopping there. Cover-and-rewrite + variant = the honest test. "Understood it" is a feeling; "rewrote it from blank" is evidence. (2 min) |
| 12 | Rule 5 — trace first | Demo the banana snippet live if there's time — most of the room predicts 3, the answer is 1. That gasp *is* the lesson: a wrong prediction found a real gap for free. (2 min) |
| 13 | Today's session | Logistics: warm-up, then 5 problems easiest→hardest. Warn them honestly: trainers will answer questions **with questions** — not to be difficult, but because handing over a step steals the rep. (2 min) |
| 14 | Beyond today | These rules apply to every course/language forever. The AI-autocomplete warning matters for this cohort: accepting Copilot/ChatGPT code you couldn't write is tutorial hell with better tooling. (2 min) |
| 15 | Close | Read it, pause, send them to the warm-up. "Timer starts now — go get stuck." (1 min) |

## Trainer conduct during the practice block

- When called over, ask to see the **plan** first. No plan → the help *is* building the plan
  together (ask them to trace one example by hand), never the code.
- Answer with questions: "What does your step 2 say?" · "What did your brain do when you solved
  the example on paper?" · "Which of the three cases is this input?"
- Enforce the timer gently but firmly; celebrate students you catch being stuck and staying with
  it. Say "good — that's the rep" out loud so the room hears it.
- End-of-session wrap (15 min): everyone closes their best solved problem and rewrites it from a
  blank file. Assign the variants as homework.
